## Summary

The **Pixell Billing System** is a comprehensive hybrid subscription platform combining base monthly plans with usage-based credit allocation.

### Core Architecture:
- **4 subscription tiers** (Free, Starter $9.99, Pro $99, Max $499.99) with tiered action credits (small/medium/large/xl)
- **6 new database tables** for subscriptions, usage tracking, credit balances, purchases, webhooks, and invoices
- **13 API endpoints** for checkout, usage management, subscription control, and Stripe webhooks
- **Stripe integration** for payment processing, trial management, and automatic billing cycles
- **Usage enforcement** at organization level with per-user analytics for visibility

### Key Features:
- **7-day trial** (no credit card required)
- **Hard usage limits** with immediate 402 errors and upgrade modals at 100% capacity
- **Auto top-up** functionality (500 credits for $20) triggered at configurable thresholds
- **Service token authentication** for orchestrator-only usage recording
- **Weekly reconciliation** (Sundays 3am) to detect and fix credit balance drift
- **Critical AWS SES emails** for payment failures, trial endings, and receipts
- **Comprehensive billing dashboard** with per-tier progress bars and per-user usage analytics
- **Failed payment handling** with 7-day grace period â†’ automatic downgrade to free tier

### Technical Highlights:
- **Idempotent operations** via idempotency keys and webhook event tracking
- **Atomic SQL updates** to prevent race conditions in concurrent usage scenarios
- **Hybrid Stripe sync** (cache frequently-accessed fields, query API for details)
- **80% credit warning** system with rate-limited notifications
- **Monthly credit reset** on billing cycle boundaries
- **Developer payout tracking** (80% revenue share - separate PAC database, out of scope)

The system is production-ready with both databases migrated, AWS Secrets Manager configured, and comprehensive error handling, monitoring, and security measures in place.
